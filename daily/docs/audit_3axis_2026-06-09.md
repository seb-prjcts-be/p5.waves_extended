## Delete-shortlist (strenge vitrine-lat)

### a_to_b — 4/3/10
API-clean and honest, but it reads as a single wiggling line plus a teaching diagram — the most boring morph the library can do, no poetry.
- code/lib: Uses the real morph signature (wave:[sine,square]+animated mix) correctly, but the result is a sine-to-square wiggle that is the dullest, most obvious morph the library offers.
- concept: Earnest didactic diagram with frozen references and a mix meter, but it is a textbook explainer, not art. Kelly/calming-device framing is aspirational not realized.

### one_or_zero — 3/5/10
Fully 3.3.0-compatible with an honest concept, but the wave engine is barely used — sign-only thresholding makes the library nearly invisible.
- code/lib: Two samplers summed and sign-thresholded. Only sign matters, so the wave is wasted — no shift/morph/wild/unpredictability. Replaceable by sin().
- concept: LeWitt/Weiner binary-rule framing is honest but well-worn; '1 or 0, the wave decides' is conceptually clean yet visually yields plain bands.

### row_plus_col — 3/6/9
Honest 3.3.0 port of a removed grid primitive, but it strands the library as a dumb lookup — no shift/morph/wild. Good concept, weak library showcase.
- code/lib: Library reduced to a stateless wave lookup; samplers created but shift/morph/wild/unpredictability all unused. Just rowV[r]+colV[c] arithmetic — any sine table would do.
- concept: Separable additive grid with live margin specimens; clean Molnar/Mohr lineage and genuinely legible axis-contribution idea. Solid but a familiar move.

### breath — 3/6/10
3.3.0-native and quietly poetic, but it renounces every signature p5.waves trick — the wave engine isn't the driver, external sines are.
- code/lib: Sampler used as a static lookup of one wave. All breathing done by external Math.sin on strokeWeight/freq, not by shift/morph/wild/range. Library adds little.
- concept: Subtraction as discipline — one wave observed precisely — is genuinely poetic and restrained, but conceptually thin and not very original as a single static trace.

### thirty-four — 4/5/10
Fully 3.3.0-compatible and clean, but a literal catalog that plots waves statically — showcases breadth, not the engine's cleverness.
- code/lib: Plots all 34 waves literally as static rows with t-drift; uses none of shift/morph/wild/range/sampler. Library is a plotter, not an expressive driver.
- concept: Honest 'library card' cataloging every waveform is tidy and useful, but it is a reference chart, not inspiring generative art.

### not_an_ocean — 4/5/10
Clean 3.3.0 code with a genuinely nice spatial morph, but its advertised SHIFT driver is smothered by Perlin noise, so the library barely shows its strength.
- code/lib: Spatial wave[1,14] morph is elegant, but the headline shift/timer output is laundered through noise() — could swap timer for noise(t) with near-identical result.
- concept: Ninety morphing wavelines as 'not an ocean' is pleasant and self-aware, but parallel-horizon-lines is a heavily worn generative trope; little fresh poetry.

## Revise (concept of code redbaar)

- **daily** (5/5/10) — Fully 3.3.0-native and tidy, but the SHIFT headline is invisible: a competent generic radial-ring plot where shift adds little legible expression.
- **daily** (5/5/10) — 3.3.0-clean shift specimen, but the binary weave threshold hides the very shift/morph it claims to sell; needs a softer mapping to earn its place.
- **sediment** (5/6/10) — Apt sediment-as-shift concept, perfectly 3.3.0-compatible, but layers are lockstep clones of one wave; fix per-layer seeds/waves to earn its place.
- **five_windows** (4/8/10) — Beautiful, well-argued domain-zoom concept, but the code barely uses p5.waves — a static lookup any sin() could do. Lean on shift/morph to earn its place.
- **rolling_shift** (6/6/10) — Fully 3.3.0-native shift specimen with a real conceptual frame, but 0.1s permanent-churn timing turns the wave-family handoff into illegible jitter across 343 cubes.
- **archive_null** (5/8/10) — Powerful, literate concept fully 3.3.0-native, but the wave engine is under-exploited — one static wave() wobbling an edge that sin/noise could nearly match.
